Buenas Prácticas al Escribir Condicionales

Saber cómo funcionan los condicionales es solo el primer paso. Escribirlos de forma clara y eficiente es lo que te diferenciará como programador. ¡Aquí tienes algunos consejos clave!

Usa el Operador Estricto (`===`)

Evita `==` para prevenir errores de conversión de tipos. El operador estricto `===` es más seguro y predecible.

Mala Práctica
if (edad == "18") { ... }
Buena Práctica
if (edad === 18) { ... }

Simplifica las Condiciones

No necesitas comparar una variable booleana con `true` o `false`. La variable por sí misma ya es una condición.

Mala Práctica
if (tieneTicket == true) { ... }
Buena Práctica
if (tieneTicket) { ... }

Evita Anidar `if` Excesivamente

Muchos `if` uno dentro de otro son difíciles de leer. Usa operadores lógicos o salidas tempranas para aplanar tu código.

Mala Práctica
if (user) {
    if (user.activo) {
        // ...
    }
}
Buena Práctica
if (!user || !user.activo) {
    return;
}
// ...

Elige `switch` para Casos Claros

Si comparas la misma variable contra múltiples valores fijos (ej. un tipo, un estado, un rol), un `switch` es más limpio que un `if-else if` largo.

Ideal para escenarios como:

  • Roles de usuario (admin, editor, etc.).
  • Días de la semana.
  • Estados de un pedido (enviado, procesando, etc.).
