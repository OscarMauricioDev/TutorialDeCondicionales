Ejemplo #3: Mensajes según Rol de Usuario con `switch`

Este ejemplo práctico demuestra cómo la estructura `switch` es ideal para manejar diferentes casos basados en un valor de texto, como puede ser el rol de un usuario en una aplicación.

Las reglas son:

  • Si el rol es "admin", se le da acceso total.
  • Si el rol es "editor", puede modificar contenido.
  • Cualquier otro rol (o ninguno) será tratado como un "invitado" con acceso limitado.

Ejemplo Visual

A continuación se muestra el código que implementa esta lógica.

Análisis: El `switch` es muy legible para este tipo de escenarios. Compara el valor de la variable `rol` con cada `case`. El bloque `default` es perfecto para manejar todos los casos no especificados, asignando un rol de invitado por defecto.
